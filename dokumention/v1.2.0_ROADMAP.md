#### Feature 1: Binary Sensors ⏱️ 45 min
**Priority:** HIGH  
**Impact:** Makes automations 10x easier

**Problem:**
Nuværende automations bruger komplekse templates:
```yaml
condition: template
value_template: >
  {% set reservations = state_attr('sensor.mielelogic_reservations', 'Reservations') %}
  {% if reservations and reservations|length > 0 %}true{% else %}false{% endif %}
```

**Solution:**
Binary sensors der returnerer `on/off`:
```yaml
condition: state
entity_id: binary_sensor.mielelogic_vaskeri_3444_has_reservation
state: 'on'
```

**Sensorer at implementere:**
1. `binary_sensor.mielelogic_vaskeri_3444_has_reservation`
   - `on`: Bruger har mindst 1 reservation
   - `off`: Ingen reservationer
   
2. `binary_sensor.mielelogic_vaskeri_3444_has_washer_reservation`
   - `on`: Har washer reservation
   - `off`: Ingen washer reservationer

3. `binary_sensor.mielelogic_vaskeri_3444_has_dryer_reservation`
   - `on`: Har dryer reservation
   - `off`: Ingen dryer reservationer

4. `binary_sensor.mielelogic_vaskeri_3444_reservation_starting_soon`
   - `on`: Reservation starter inden for 15 minutter
   - `off`: Ingen reservation snart
   - Attribute: `minutes_until_start` (number)

5. `binary_sensor.mielelogic_vaskeri_3444_washer_available`
   - `on`: Mindst 1 washer er ledig
   - `off`: Alle washers optaget

6. `binary_sensor.mielelogic_vaskeri_3444_dryer_available`
   - `on`: Mindst 1 dryer er ledig
   - `off`: Alle dryers optaget

**Implementation:**

**Reservations Sensor:**
```python
@property
def extra_state_attributes(self):
    reservations = self.coordinator.data.get("reservations", {}).get("Reservations", [])
    
    # Find next reservation
    now = datetime.now()
    upcoming = [r for r in reservations if datetime.fromisoformat(r["Start"]) > now]
    upcoming.sort(key=lambda r: r["Start"])
    next_reservation = upcoming[0] if upcoming else None
    
    return {
        "total_count": len(reservations),
        "washer_count": len([r for r in reservations if r.get("MachineType") in ["51", "85"]]),
        "dryer_count": len([r for r in reservations if r.get("MachineType") == "58"]),
        "next_reservation": {
            "machine_name": next_reservation.get("MachineName"),
            "start_time": next_reservation.get("Start"),
            "end_time": next_reservation.get("End"),
            "duration_minutes": next_reservation.get("Duration"),
        } if next_reservation else None,
        "reservations_today": len([r for r in reservations if is_today(r["Start"])]),
        "raw_data": reservations,  # Keep for backwards compatibility
    }
```

**Machine Status Sensors:**

```python
@property
def extra_state_attributes(self):
    return {
        "machine_number": self._machine.get("MachineNumber"),
        "unit_name": self._machine.get("UnitName"),
        "machine_type": self._machine.get("MachineType"),
        "machine_type_name": self._get_machine_type_name(),  # "Washer" / "Dryer"
        "is_available": self._is_available(),
        "is_reserved": self._is_reserved(),
        "is_running": self._is_running(),
        "status_text": self._machine.get("Text1"),
        "reservation_info": self._machine.get("Text2"),
        "color_code": self._machine.get("MachineColor"),
        "symbol_code": self._machine.get("MachineSymbol"),
    }

def _get_machine_type_name(self):
    machine_type = self._machine.get("MachineType")
    if machine_type in ["51", "85"]:
        return "Washer"
    elif machine_type == "58":
        return "Dryer"
    return "Unknown"

def _is_available(self):
    status = self._machine.get("Text1", "").lower()
    return "ledig" in status or "available" in status

def _is_reserved(self):
    status = self._machine.get("Text1", "").lower()
    return "optaget" in status or "reserved" in status

def _is_running(self):
    status = self._machine.get("Text1", "").lower()
    return "kører" in status or "running" in status
```

#### Feature 3: Response Caching ⏱️ 30 min
**Priority:** HIGH  
**Impact:** Reduces API load by 90% on HA restarts

**Problem:**
Ved HA genstart laver hver entity fetch = 20+ API calls på 10 sekunder.
MieleLogic API kan rate limite (30 min cooldown).

**Solution:**
Cache responses i coordinator med TTL (time-to-live).

**Implementation i `coordinator.py`:**
```python
from datetime import datetime, timedelta
from zoneinfo import ZoneInfo

class MieleLogicDataUpdateCoordinator(DataUpdateCoordinator):
    def __init__(self, hass: HomeAssistant, config_entry):
        # ... existing code ...
        
        # Add cache storage
        self._cache = {}
        self._cache_ttl = timedelta(seconds=60)  # 60 second cache
        
    def _get_cache_key(self, endpoint: str) -> str:
        """Generate cache key for endpoint."""
        return f"{endpoint}_{self.laundry_id}"
    
    def _is_cache_valid(self, cache_key: str) -> bool:
        """Check if cached data is still valid."""
        if cache_key not in self._cache:
            return False
        
        cached_data = self._cache[cache_key]
        cache_time = cached_data.get("timestamp")
        
        if not cache_time:
            return False
        
        age = datetime.now(ZoneInfo("UTC")) - cache_time
        return age < self._cache_ttl
    
    def _get_from_cache(self, cache_key: str):
        """Get data from cache."""
        if self._is_cache_valid(cache_key):
            _LOGGER.debug("Cache HIT for %s (age: %s)", cache_key, 
                         datetime.now(ZoneInfo("UTC")) - self._cache[cache_key]["timestamp"])
            return self._cache[cache_key]["data"]
        
        _LOGGER.debug("Cache MISS for %s", cache_key)
        return None
    
    def _save_to_cache(self, cache_key: str, data: dict):
        """Save data to cache with timestamp."""
        self._cache[cache_key] = {
            "data": data,
            "timestamp": datetime.now(ZoneInfo("UTC"))
        }
        _LOGGER.debug("Cached data for %s", cache_key)
    
    async def _fetch_with_cache(self, url: str, cache_key: str, headers: dict) -> dict:
        """Fetch data with caching."""
        # Try cache first
        cached = self._get_from_cache(cache_key)
        if cached is not None:
            return cached
        
        # Cache miss - fetch from API
        async with aiohttp.ClientSession() as session:
            async with session.get(url, headers=headers) as response:
                # ... existing error handling ...
                data = await response.json()
                self._save_to_cache(cache_key, data)
                return data
```

**Usage in `_async_update_data`:**
```python
async def _async_update_data(self):
    # ... token refresh logic ...
    
    data = {}
    
    # Fetch reservations (with cache)
    url = f"https://api.mielelogic.com/v7/reservations?laundry={self.laundry_id}"
    cache_key = self._get_cache_key("reservations")
    data["reservations"] = await self._fetch_with_cache(url, cache_key, headers)
    
    # Fetch machine states (with cache)
    url = f"https://api.mielelogic.com/v7/Country/DA/Laundry/{self.laundry_id}/laundrystates?language=da"
    cache_key = self._get_cache_key("machine_states")
    data["machine_states"] = await self._fetch_with_cache(url, cache_key, headers)
    
    # Fetch account details (with cache)
    url = "https://api.mielelogic.com/v7/accounts/Details"
    cache_key = self._get_cache_key("account")
    data["account_details"] = await self._fetch_with_cache(url, cache_key, headers)
    
    return data
```

**Benefits:**
- HA restart: 3 API calls instead of 20+
- 60 second cache reduces redundant calls
- Graceful degradation (cache miss = normal fetch)
- Debug logging shows cache hit/miss

#### Feature 4: Rate Limiting Handler ⏱️ 20 min
**Priority:** MEDIUM  
**Impact:** Prevents integration breaking on rate limit

**Problem:**
MieleLogic API returns 429 (Too Many Requests) = 30 min cooldown.
Integration crashes and requires manual reload.

**Solution:**
Detect 429 errors, pause updates gracefully, show user-friendly message.

**Implementation i `coordinator.py`:**
```python
from homeassistant.helpers import issue_registry as ir

class MieleLogicDataUpdateCoordinator(DataUpdateCoordinator):
    def __init__(self, hass: HomeAssistant, config_entry):
        # ... existing code ...
        self._rate_limited_until = None
        
    async def _async_update_data(self):
        """Fetch data with rate limit handling."""
        
        # Check if we're currently rate limited
        if self._rate_limited_until:
            now = datetime.now(ZoneInfo("UTC"))
            if now < self._rate_limited_until:
                remaining = (self._rate_limited_until - now).total_seconds() / 60
                _LOGGER.warning(
                    "Rate limited by MieleLogic API. Skipping update. "
                    "Retry in %.1f minutes", remaining
                )
                # Return last known data
                return self.data if self.data else {}
            else:
                _LOGGER.info("Rate limit expired, resuming normal updates")
                self._rate_limited_until = None
        
        try:
            # ... existing fetch logic ...
            
        except aiohttp.ClientResponseError as err:
            if err.status == 429:
                # Rate limited!
                self._handle_rate_limit()
                raise UpdateFailed(
                    "Rate limited by MieleLogic API. Updates paused for 30 minutes. "
                    "This is a safety measure by the API provider."
                )
            else:
                raise
    
    def _handle_rate_limit(self):
        """Handle rate limit by pausing updates."""
        self._rate_limited_until = datetime.now(ZoneInfo("UTC")) + timedelta(minutes=30)
        
        _LOGGER.error(
            "RATE LIMITED by MieleLogic API! Updates paused until %s. "
            "This typically happens when making too many requests. "
            "The integration will automatically resume after 30 minutes.",
            self._rate_limited_until
        )
        
        # Create a repair issue in HA
        ir.async_create_issue(
            self.hass,
            DOMAIN,
            "rate_limited",
            is_fixable=False,
            severity=ir.IssueSeverity.WARNING,
            translation_key="rate_limited",
            translation_placeholders={
                "retry_time": self._rate_limited_until.strftime("%H:%M"),
            },
        )
```

#### Feature 5: Machine Sensor Cleanup ⏱️ 25 min
**Priority:** MEDIUM  
**Impact:** Cleaner entity registry

**Problem:**
Hvis en maskine fjernes fra vaskeriet, forbliver sensoren i HA.

**Solution:**
Track hvilke maskiner der eksisterer, fjern sensorer for missing machines.

**Implementation i `sensor.py`:**
```python
async def async_setup_entry(hass, config_entry, async_add_entities):
    """Set up the sensor platform."""
    coordinator = hass.data[DOMAIN][config_entry.entry_id]
    
    # Global sensors
    sensors = [
        MieleLogicReservationsSensor(coordinator, config_entry),
        MieleLogicWasherStatusSensor(coordinator, config_entry),
        MieleLogicDryerStatusSensor(coordinator, config_entry),
        MieleLogicAccountSensor(coordinator, config_entry),
    ]
    
    # Track machine sensors
    if not hasattr(coordinator, "_machine_sensors"):
        coordinator._machine_sensors = {}
    
    # Add/update machine sensors
    machine_states = coordinator.data.get("machine_states", {}).get("MachineStates", [])
    for machine in machine_states:
        machine_id = machine.get("MachineNumber")
        if machine_id not in coordinator._machine_sensors:
            sensor = MieleLogicMachineStatusSensor(coordinator, config_entry, machine)
            coordinator._machine_sensors[machine_id] = sensor
            sensors.append(sensor)
    
    async_add_entities(sensors)

async def async_unload_entry(hass: HomeAssistant, entry: ConfigEntry) -> bool:
    """Handle removal of an entry."""
    coordinator = hass.data[DOMAIN][entry.entry_id]
    
    # Clean up tracked machine sensors
    if hasattr(coordinator, "_machine_sensors"):
        coordinator._machine_sensors.clear()
    
    return True
```

**Benefits:**
- Entity registry stays clean
- No orphaned sensors
- Automatic cleanup on unload
